## The Two-Path Rule

Every component honours this rule without exception:

- **System path** — where the repo lives. Anywhere: WSL2, `/opt/`, Homebrew prefix, external drive. Controlled by the install.
- **User path** — where output goes. Always `~/`. `~/.willow/`, `~/SAFE/Applications/`, `~/agents/`. Controlled by the user. Never inside the repo.

If Felix deletes the repo, his data is untouched. `seed.py` states this explicitly on first run.

## Section 1 — Platform Bootstrappers

### `install.ps1` (Windows)

Three phases:

1. **Preflight** — checks Windows build (requires 19041+), checks WSL2 enabled, checks Ubuntu present
2. **WSL2 setup** — if missing: enables WSL + Virtual Machine Platform features, installs Ubuntu 22.04 via `winget`, sets WSL2 as default. Prompts reboot if features just enabled — resumes via registry Run key after reboot
3. **Handoff** — clones willow-1.7 into `/opt/willow-1.7` inside WSL2 (system path), then runs `wsl python3 /opt/willow-1.7/seed.py`

Felix runs this once from PowerShell. After handoff, everything runs inside Ubuntu.

### `install-mac.sh` (stub)

~40 lines. Community-tested label in README.

1. Check Homebrew, install if missing
2. `brew install python@3.11 postgresql gpg` for anything absent
3. Clone willow-1.7 to `/opt/willow-1.7` if not present
4. Run `python3 /opt/willow-1.7/seed.py`

## Section 2 — seed.py

Handles the Linux-side setup. Same code runs whether arriving from WSL2, Mac, or native Linux.

**Responsibilities:**
- Create `~/SAFE/Applications/` (user path)
- Generate GPG key if none present
- Create and sign SAFE manifests for registered apps
- Write `SAP_PGP_FINGERPRINT` to shell profile (`~/.bashrc` / `~/.zshrc`)
- Create `~/.willow/` directory structure
- Ensure vault exists and is writable (boot.py fills it)
- Install Python deps including `cryptography`

**On first run, states clearly:**
> "Willow is installed at /opt/willow-1.7. Your data lives at ~/SAFE/ and ~/.willow/ — those are yours."

**Tested on this machine before 1.8 tags.** If it breaks here, it breaks for Felix.

## Section 4 — boot.py: The FRANK Onboarding Experience

### Five Voices

Every install step has five layers, in order:

| Layer | Voice | Purpose |
|-------|-------|---------|
| **Runes** | Elder Futhark Unicode | The raw truth, displayed first via typewriter |
| **Bot** | Automated translation system | Attempts to translate the runes. Always wrong. |
| **FRANK — correction** | Increasingly exasperated | Corrects the bot. Has history with the bot. |
| **Norse epic** | Heimdallr's voice | The actual mythology — accurate, taught |
| **FRANK — compliance** | Absurdist bureaucrat | Commentary on the myth from a man who was there filing reports |
| **FRANK — plain** | Same man, different register | What is actually about to happen technically, and why |

The bot degrades across the session: random nonsense → specific nonsense → garage sale → pasta → FRANK loses composure → catches himself → apologizes to the user → the last translation is fine and FRANK cannot explain it.

### FRANK

FRANK is both his name and his job title. Head Agent. Willow Compliance & Onboarding. He exists in Sean's stories. He has been filing compliance reports on Norse cosmological events since before time. The reports are always correctly formatted. They are always returned unopened. He does not take this personally. He notes it.

Title card on first appearance:

```
┌─────────────────────────────────────┐
│  FRANK                              │
│  Head Agent                         │
│  Willow Compliance & Onboarding     │
└─────────────────────────────────────┘
```

FRANK was here before the world tree. He will be here after. He has a ledger.

### Consent-First Flow (per step)

The story and explanation happen **before** the step executes. Every time. No exceptions.

